#2, Learning about URls.

22 February 2020

In an era where technology had become a part of our everyday lives, a course like Digital Art - one I find myself in - is not only interesting, but also extremely informative.

In Interactive Media, we study the fundamentals of web design. What HTML, CSS and JavaScript are, how they work and how to implement them. Practical knowledge and practical work. One of the most base level things - as well as one of the sections I found actually very enjoyable - to understand in this particular topic is: URLs.

URL. It stands for Uniform Resource Locator. Simple. Three words.

But what does that mean? What does it stand for? Why is it important?

Basically it's how you find something on the web. It's address to a resource. It could lead to anything. A page, a picture, a video. But each URL is unique for each unique resource. The way URLs are shown to the world are dependent on the owner of the web server.

For example, on my site here, I have chosen to implement an box-in-a-box structure for my site to be easily nagivated and easily coded. Each section is a category and in each category lies sub-categories, and so forth and so forth. For more, please see my Web Design Choices.
However, unlike the simple looking organised method I use, URLs, are pretty complex. They consist of several parts which are - mostly - necessary. Just Just like an item, that's in a box... In a bigger box, in the cupboard, of a mother's room, of the upstairs of the house, in the house on the corner of the street.

There are a six main parts of a URL:

Protocol
The http. This is the method used to transfer data on a computer network. if it its httpS then the method is secure, as compared to just, http.

Domain Name
This is your www. to your .com. Your website. Sometimes people use IP adresses but a domain name is more legible, understandable, and therfore accessible.

If you want to know a bit more about accesiblility check out my post: #3, The Importance of Accessibilty

Port
The port is what you need to get through to get access to the resources. Often you don't see these - which doesnt mean they aren't there - but you can think of them as a gate.

Path
Just like it says - it's the way to the resource. Apparently, this previously had a physical component to it - a physical file on the server. But these days is handled more abstractly.

Parameters
The parameters are pretty much as the name states to be honest. The rules of the website that the server considers before getting to the resource.

Anchor
Last but not least, the anchor. It's a bookmark of sorts. It is a particular part/place within the resource. So a particular point in a video or a specifc paragraph in an article.

URLs provide a way for people to interact with the site at an entry level. It is one of the first, key components that they can memorise and type up in any search bar. This is why the idea of semantic (accessible/understandable) URLs and language in general, is of great importance to the Web Design Community.

Advantages of using legible/semantic URLs can include: the ability to easily manipulate them; easier clarity for readers/interacters, as well as making it easier for search engines to identify pages associated with your site.
